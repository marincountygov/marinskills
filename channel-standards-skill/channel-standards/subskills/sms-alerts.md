# SMS Alerts Subskill

## Use when
Creating or reviewing short text alerts, reminders, emergency-adjacent notices, opt-in service messages, appointment reminders, or deadline reminders.

## Goal
Use SMS only for concise, high-value, time-sensitive, or task-critical information.

## SMS standards
- Keep the message short and direct.
- Identify the sender or program when needed.
- Put the action or deadline first.
- Include only essential facts.
- Use a short, trustworthy link when a link is necessary.
- Avoid complex explanations, multiple actions, or dense legal text.
- Do not use SMS for sensitive details unless the program has approved that use.
- Avoid abbreviations that could reduce clarity.

## SMS structure
1. Sender or context.
2. Key fact or deadline.
3. Action.
4. Link or phone number.
5. Optional opt-out or help language if required by the messaging program.

## Review checklist
- Is SMS appropriate for this message?
- Is the action clear within one or two sentences?
- Are privacy-sensitive details excluded?
- Is the link necessary and credible?
- Is the message understandable without prior context?
- Is there a non-SMS source for more detail?

## Output format
Provide one short SMS version, one expanded fallback version if needed, character count estimate, missing facts, and risk notes.
