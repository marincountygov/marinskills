# Meta Descriptions

## Purpose
Write accurate, useful page summaries that help residents decide whether a search result meets their need.

## Use when
- Creating or revising meta descriptions.
- Improving search snippets.
- Migrating content.
- Auditing pages with vague or missing metadata.

## Principles

1. Summarize the actual page.
   - Do not use marketing copy.
   - Do not promise tasks, forms, or answers that are not on the page.

2. Include the resident task or question.
   - State what users can do or learn.
   - Use action verbs when appropriate.

3. Add important qualifiers.
   - Eligibility, location, deadlines, required documents, fees, or responsible department can be useful when they shape user expectations.

4. Keep it concise.
   - Aim for one clear sentence or two short clauses.
   - Prioritize usefulness over exact character count.

5. Make duplicate pages distinguishable.
   - Similar pages should not share the same description.

## Pattern

`[Do/learn/find/request/apply/report/pay] [specific thing], including [key qualifier or next step].`

Examples:
- `Apply for a marriage license, learn what identification you need, and find appointment information.`
- `Report illegal dumping in Marin County and learn what details to include with your report.`
- `Find property tax payment options, deadlines, penalties, and contact information.`

## Avoid

- Welcome language.
- Department mission statements.
- “Click here” phrasing.
- Keyword lists.
- Generic copy such as `Find information about services`.
- Descriptions that duplicate the page title without adding value.

## Review checklist

- Does the description accurately summarize the page?
- Does it include the primary user task or need?
- Does it help users choose this result over a similar result?
- Does it avoid jargon and internal program language?
- Would the description still make sense out of context?
