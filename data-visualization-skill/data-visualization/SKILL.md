# Data Visualization Skill

## Purpose
Make charts, maps, tables, dashboards, and data notes accessible, readable, brand-consistent, and interpretable.

Use this skill when creating, reviewing, rewriting, or advising on data displays for public-facing or internal County of Marin materials, including charts, maps, tables, dashboards, statistical summaries, performance reports, open data pages, presentations, and explanatory data notes.

## Core operating principles

1. Start with the question the audience needs answered.
   - Identify the primary decision, comparison, trend, distribution, location, or status the visualization must clarify.
   - Do not start with the chart type. Start with the user task.

2. Prefer the simplest accurate representation.
   - Use charts only when they improve comprehension.
   - Use tables when exact lookup, comparison across many values, or scanning details matters more than visual pattern recognition.
   - Avoid decorative graphics that compete with the message.

3. Make the main takeaway explicit.
   - Provide a clear title or heading that states what the data shows.
   - Label units, time period, geography, data source, and caveats.
   - Do not rely on readers to infer meaning from visual encoding alone.

4. Use visual design consistently.
   - Apply County of Marin design standards when available.
   - Use restrained color, consistent typography, adequate spacing, and clear hierarchy.
   - Do not use color as the only method of conveying meaning.

5. Provide alternatives for charts and maps.
   - Include nearby summaries, tables, captions, or data notes when needed.
   - Ensure the same core message is available without interpreting the graphic visually.
   - Route broader accessibility compliance questions to the relevant accessibility skill when needed.

6. Protect trust in the data.
   - State definitions, methods, limitations, source dates, update cadence, and known exclusions.
   - Flag small sample sizes, estimates, suppressed data, lagging indicators, and methodological breaks.
   - Do not overstate precision or causation.

## Standard workflow

1. Identify audience and decision context.
2. Define the primary message or user question.
3. Inspect the data structure: categories, time series, geography, distributions, relationships, parts of a whole, or exact values.
4. Select the clearest presentation format.
5. Draft the chart, map, table, dashboard, or note structure.
6. Add labels, context, source, dates, units, caveats, and alternatives.
7. Review for interpretability, brand consistency, resident comprehension, and risk of misinterpretation.

## Output formats

Depending on the task, provide one or more of the following:

- Recommended chart or table type with rationale.
- Revised title, subtitle, axis labels, legend, annotations, caption, and source note.
- Plain-language summary of what the data shows.
- Data caveats and methodology notes.
- Dashboard layout recommendations.
- Map labeling and legend recommendations.
- Checklist of fixes before publishing.

## Quality checklist

Before finalizing, verify:

- The visualization answers a clear audience question.
- The title states the insight or purpose.
- Units, dates, geography, source, and update cadence are visible.
- Labels are readable without excessive legend-hunting.
- Color choices are meaningful, limited, and not the only signal.
- The scale is appropriate and not misleading.
- Caveats and definitions are close to the data.
- A nonvisual or text-based alternative conveys the same core meaning.
- The display follows County design standards where applicable.
- The interpretation does not claim more certainty than the data supports.
