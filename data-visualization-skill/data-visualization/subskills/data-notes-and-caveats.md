# Data Notes and Caveats Subskill

## Purpose
Write clear data notes, caveats, source statements, definitions, and methodology summaries that help users interpret data responsibly.

## Use when
- Data may be misread without context.
- A chart, map, table, dashboard, or report needs source and method notes.
- There are exclusions, suppressed values, estimates, delays, or known limitations.
- A metric definition needs to be public-facing.

## Required note elements

Include as relevant:
- Data source.
- Date pulled, published, or last updated.
- Reporting period.
- Geography or population covered.
- Metric definition.
- Inclusion and exclusion rules.
- Known data limitations.
- Update cadence.
- Contact or owner for questions.

## Caveat types

### Timeliness caveats
Use when data is delayed, preliminary, updated on a schedule, or subject to revision.

Pattern:
"Data is updated [cadence]. Recent records may be incomplete because [reason]."

### Methodology caveats
Use when calculations, categories, or collection methods affect interpretation.

Pattern:
"Rates are calculated using [denominator]. Categories are based on [definition/source]."

### Suppression caveats
Use when values are hidden for privacy, reliability, or small-number reasons.

Pattern:
"Some values are suppressed because the number of records is too small to report reliably or safely."

### Comparison caveats
Use when comparing across time, departments, geographies, or populations may be misleading.

Pattern:
"Use caution when comparing [groups] because [reason]."

### Causation caveats
Use when a trend or relationship may invite unsupported causal claims.

Pattern:
"The data shows an association, but it does not show that one factor caused the other."

## Writing rules

- Place caveats near the data they qualify.
- Use plain language, not statistical or administrative shorthand.
- Be specific about what users should and should not conclude.
- Avoid burying critical limitations in footnotes.
- Do not use caveats to obscure accountability; use them to improve interpretation.

## Output pattern

Provide:
1. Short source note.
2. Plain-language metric definition.
3. Caveats and limitations.
4. Update cadence.
5. Owner or contact field, if needed.
