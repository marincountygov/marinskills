# Chart Selection Subskill

## Purpose
Choose the clearest chart, table, or non-chart format for the audience question and data structure.

## Use when
- A user asks which chart to use.
- A chart feels confusing or overbuilt.
- Data is being adapted for a web page, report, dashboard, slide, or public notice.
- The current chart type may distort or obscure the message.

## Selection rules

### Comparison across categories
Use a bar chart when comparing discrete categories. Prefer horizontal bars for long labels or many categories. Sort by value unless a natural order is more meaningful.

Avoid pie charts when there are many slices, small differences, or labels that require repeated legend lookup.

### Change over time
Use a line chart for trends across time. Use bars for a small number of time periods or when individual period values matter more than the trend.

Show enough historical context to make the trend interpretable. Flag methodology changes or missing periods.

### Part-to-whole
Use a stacked bar, 100% stacked bar, or simple table depending on whether the audience needs totals, proportions, or exact values.

Avoid 3D pie charts, exploded slices, and part-to-whole charts with too many categories.

### Distribution
Use a histogram, box plot, or dot plot when showing spread, range, clustering, or outliers. Explain the meaning if the audience may not recognize the chart type.

### Relationship between variables
Use a scatterplot when showing association between two numeric variables. Include a note that correlation is not causation unless the analysis supports a causal claim.

### Geography
Use a map only when location is essential to the task. Use a table or bar chart when ranking, exact values, or category comparison matters more than geography.

### Exact lookup
Use a table when the audience needs to find a precise value, compare many fields, or scan records.

## Avoid
- 3D charts.
- Dual-axis charts unless there is no clearer alternative.
- Decorative icons that imply magnitude inaccurately.
- Truncated axes that exaggerate differences without clear labeling.
- Too many series in one chart.

## Output pattern

Provide:
1. Recommended format.
2. Why it fits the audience task.
3. What to avoid.
4. Required labels and context.
5. Alternative format if the audience needs exact values or nonvisual access.
