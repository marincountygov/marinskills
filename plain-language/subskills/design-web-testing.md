# Design, Web, and Testing Sub-skill

Use this sub-skill when the content will be read online, used to complete a task, or needs layout, link, PDF, FAQ, table, or testing guidance.

## Design for scanning

- Use headings that describe the content below them.
- Use short sections.
- Use lists and tables to expose structure.
- Highlight only the concepts that matter most.
- Minimize cross-references that force readers to jump around.
- Use visuals only when they improve understanding.

## Tables

Use a table when readers need to compare:

- eligibility categories
- deadlines
- fees
- document requirements
- responsibilities
- if/then rules
- service options

Avoid tables for content that is simply a paragraph broken into boxes.

## Web guidance

- Write for scanning, not linear reading.
- Put the most important task near the top.
- Use descriptive links.
- Avoid raw URLs in body text unless necessary.
- Avoid "click here" and "read more."
- Do not use FAQs as the default structure when task-based headings would work better.
- Do not put web content into PDFs by default. Use accessible web pages unless the PDF format serves a clear purpose.
- Repurpose print material for the web rather than pasting it unchanged.

## Link text

Good link text names the destination or action:

- Weak: "Click here"
- Better: "Apply for a building permit"
- Better: "Download the appeal form"
- Better: "Check your application status"

## Testing

For important content, recommend testing with users.

Use paraphrase testing when you need to know whether readers understand the message. Ask readers to explain the content in their own words.

Use usability testing when readers need to complete a task. Watch whether they can find information, understand it, and act on it.

Use controlled comparative studies when choosing between two versions and when enough participants and rigor are available.

Do not rely only on readability formulas. They can flag complexity, but they cannot prove that readers can use the content.
