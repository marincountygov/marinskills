# Organization Sub-skill

Use this sub-skill when content is hard to follow, buries the lead, lacks headings, mixes topics, or puts conditions before the main rule.

## Main rule

Organize around the reader's task. Put the main idea before exceptions, conditions, background, and procedural detail unless the condition controls whether the reader should continue.

## Techniques

- Start with the bottom line.
- Use informative headings that answer reader questions.
- Put one topic in each section.
- Start paragraphs with topic sentences.
- Use transition words to show sequence, contrast, cause, or result.
- Use lists for steps, requirements, documents, conditions, exceptions, and choices.
- Number steps when order matters.
- Use bullets when order does not matter.

## List rules

- Introduce every list with a useful lead-in sentence.
- Keep list items parallel.
- Do not bury long sentences inside bullets.
- Do not overuse lists for trivial points.
- Use tables when readers need to compare options, conditions, deadlines, or responsibilities.

## Common structural patterns

### Notice or letter

1. What happened or what we decided
2. What it means for you
3. What you need to do
4. Deadline
5. How to get help
6. Appeal or review rights, if any

### Instructions

1. Who can do this
2. What you need before you start
3. Steps
4. What happens next
5. Problems or exceptions
6. Contact

### Web page

1. Page title based on the user task
2. One-sentence purpose statement
3. Eligibility or audience fit
4. Main action
5. Supporting details
6. Contact or next step
