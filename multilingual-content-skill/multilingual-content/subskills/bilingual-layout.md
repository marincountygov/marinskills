# Bilingual Layout Subskill

## Purpose
Structure bilingual and multilingual content so residents can find the right language, compare information when needed, and complete tasks without confusion.

## Use when
Use this subskill when planning pages, documents, notices, forms, flyers, service pages, or transaction flows in more than one language.

## Layout models

### Separate language pages
Use when:

- Content is long or complex
- Each language needs its own metadata, search snippet, or URL
- Updates must be tracked independently
- The page includes multiple sections, forms, or service steps

### Side-by-side bilingual layout
Use when:

- Content is short
- Direct comparison is useful
- Space is controlled, such as flyers, notices, signs, or short instructions

### Stacked bilingual layout
Use when:

- Mobile readability matters
- The content is short to medium length
- Residents should encounter both languages without switching pages

### Language selector or landing page
Use when:

- Multiple languages are available
- The task is important enough to support explicit language selection
- Users may arrive from search or shared links

## Layout checks

- Make the language choice visible near the top.
- Keep equivalent content synchronized.
- Avoid mixing languages within the same sentence unless using an official term or proper noun.
- Keep contact information and deadlines identical across languages.
- Do not force residents to use English to navigate to non-English content.
- Preserve reading order and mobile usability.
- Label downloadable files by language.

## Output pattern

Provide:

1. **Recommended layout model** — separate pages, side-by-side, stacked, or selector.
2. **Reason** — task complexity, content length, maintenance, and resident use.
3. **Structure** — suggested headings or section order.
4. **Synchronization risks** — items likely to diverge across languages.
5. **Publishing notes** — URLs, titles, metadata, file labels, or update handling.
