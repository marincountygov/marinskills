# Duplication and Canonical Pages Subskill

## Purpose

Identify overlapping content, select canonical pages, consolidate duplicates, and define redirects or retirement actions so users and search systems have one authoritative source for each need.

## Use this subskill when

- Multiple pages answer the same user question
- A document, page, department page, and service page contain overlapping instructions
- Search results show competing pages
- Content owners disagree about which page should be authoritative
- A site migration requires consolidation or redirect planning

## Duplicate types

Classify duplication before recommending action:

- **Exact duplicate**: same or nearly identical content in multiple places
- **Partial duplicate**: overlapping sections with some unique details
- **Competing page**: different pages target the same user intent
- **Legacy page**: outdated page still accessible or indexed
- **Document duplicate**: PDF or file repeats page content
- **Ownership duplicate**: multiple departments publish separate versions of shared guidance

## Canonical page criteria

Choose the canonical page based on:

1. Best match to primary user intent
2. Clear ownership and maintenance accountability
3. Most complete and current information
4. Best placement in the site structure
5. Strongest inbound links or public recognition, if known
6. Lowest risk of breaking essential user journeys

## Recommended actions

Use one of these dispositions:

- Keep as canonical
- Merge into canonical page
- Rewrite as a supporting page
- Redirect to canonical page
- Archive or unpublish
- Keep but clearly differentiate purpose
- Replace document content with an HTML page and retain file only if legally or operationally required

## Output format

Provide a duplication map with:

1. Page or file name
2. URL or location, if known
3. Duplicate type
4. User intent served
5. Recommended disposition
6. Canonical destination
7. Redirect or link action
8. Owner and review notes

## Review checklist

- Is there one authoritative page for each user intent?
- Are duplicates removed, redirected, or clearly differentiated?
- Are document versions controlled?
- Are old links and search results handled?
- Is ownership clear for future updates?
