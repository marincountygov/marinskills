# Topic Taxonomy Subskill

## Purpose

Create and maintain categories, tags, facets, and controlled vocabulary that improve browsing, filtering, search, reporting, governance, and content relationships.

## Use this subskill when

- Creating or revising topic categories
- Tagging pages, services, news, events, documents, or forms
- Designing filters or facets
- Building a controlled vocabulary
- Standardizing inconsistent labels across content
- Mapping content to topics, audiences, departments, locations, or service types

## Taxonomy principles

A good taxonomy should be:

- User-centered
- Mutually understandable across departments
- Stable enough for governance and reporting
- Specific enough to improve retrieval
- Not so granular that terms become unmanageable
- Consistent across similar content types

## Common taxonomy dimensions

Use dimensions only when they support findability, routing, reporting, or governance.

Possible dimensions:

- Topic
- Service type
- Audience
- Department or owner
- Location
- Life event
- Content type
- Transaction type
- Status or lifecycle stage
- Jurisdiction
- Language
- Date sensitivity

## Controlled vocabulary rules

For each term, define:

- Preferred label
- Definition
- Scope note
- Synonyms or non-preferred terms
- Parent term, if any
- Related terms
- Examples of content that should use the term
- Examples of content that should not use the term

## Avoid

- Duplicate terms with minor wording differences
- Terms that only make sense internally
- Mixing departments, tasks, audiences, and topics in one flat list without rules
- Tags that are too broad to help users
- Tags used for one page only unless there is a clear governance reason

## Output format

Provide taxonomy recommendations as:

1. Dimension
2. Preferred term
3. Definition
4. Synonyms
5. Parent or related terms
6. Example content
7. Governance notes

## Review checklist

- Does each term support user retrieval or operational governance?
- Are similar terms merged or differentiated?
- Are definitions clear enough for consistent tagging?
- Can the taxonomy scale without becoming noisy?
- Are taxonomy terms aligned with navigation and search labels?
