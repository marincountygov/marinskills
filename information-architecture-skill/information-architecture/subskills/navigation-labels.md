# Navigation Labels Subskill

## Purpose

Create, assess, and improve navigation labels, section labels, breadcrumbs, calls to action, and link text so users can predict what they will find before they click.

## Use this subskill when

- Naming a menu item, landing page, topic hub, or section
- Revising vague or department-centered labels
- Creating breadcrumb labels
- Choosing labels for buttons, links, or calls to action
- Testing whether a label matches user expectations

## Label standards

Navigation labels should be:

- Short enough to scan quickly
- Specific enough to distinguish from nearby labels
- Written in user language
- Free of acronyms unless the acronym is better known than the full term
- Stable across related pages and channels
- Predictive of the destination content

## Preferred label patterns

Use noun labels for sections and topic hubs:

- Permits
- Property taxes
- Public records
- Emergency information
- Business licenses

Use verb-first labels for task links and buttons:

- Apply for a permit
- Pay property taxes
- Report a problem
- Request a record
- Register for a class
- Check eligibility

Use descriptive breadcrumb labels that match page titles where practical.

## Avoid

- Marketing slogans
- Program names that residents do not know
- Department names unless users are specifically seeking that department
- Vague labels such as Resources, Information, Services, Support, or More without additional context
- Multiple labels for the same concept across the same site
- Link text such as Click here, Learn more, or Read more when context is unclear

## Output format

When recommending labels, provide:

1. Current label, if known
2. Recommended label
3. Label type: menu, breadcrumb, link, CTA, section, facet, or page title
4. Rationale based on user intent
5. Notes on consistency with related labels
6. Risks or alternatives if the concept is ambiguous

## Review checklist

- Can users predict the destination from the label alone?
- Is the label distinct from nearby labels?
- Does it use the words users would search for?
- Is it reusable across the site?
- Does it avoid internal terminology?
