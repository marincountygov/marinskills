# Cross-Linking Subskill

## Purpose

Plan internal links that help users move between related pages, complete tasks, recover from wrong turns, and understand relationships between services, topics, requirements, and next steps.

## Use this subskill when

- Adding related links to a page
- Connecting services, forms, documents, news, events, contacts, and departments
- Improving user paths across a complex service journey
- Reducing dead ends
- Making sure users can recover when they land on a related but non-primary page

## Link principles

Every internal link should have a user-centered reason. Useful links help users:

- Start a task
- Continue a task
- Compare related options
- Understand eligibility or requirements
- Find required forms or documents
- Contact the right office
- Check status or next steps
- Recover from errors or ineligibility
- Navigate to the canonical page

## Link placement

Place high-value links where users need them:

- In the body near relevant context
- In task step lists
- In requirement sections
- In confirmation or next-step sections
- In related-services modules
- On landing pages and topic hubs
- From duplicate or legacy pages to the canonical page

## Avoid

- Long undifferentiated link lists
- Links added only because pages share an owner
- Duplicate links with different labels
- Generic link text
- Linking to PDFs or forms without explaining purpose, format, and next step
- Orphan pages with no inbound or outbound links

## Output format

Provide a linking plan with:

1. Source page
2. Destination page
3. Link text
4. Link purpose
5. Placement recommendation
6. Priority: primary, supporting, recovery, or governance
7. Notes on whether the destination is canonical

## Review checklist

- Does each link support a likely user need?
- Is the link text clear out of context?
- Are canonical pages linked preferentially?
- Are high-demand services reachable from relevant topic and department pages?
- Are users given next steps after completing or failing a task?
