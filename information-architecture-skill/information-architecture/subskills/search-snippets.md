# Search Snippets Subskill

## Purpose

Write and evaluate search result titles, summaries, descriptions, and snippets so users can choose the right result from internal search or external search.

## Use this subskill when

- Writing page titles or meta descriptions
- Improving internal search result summaries
- Differentiating similar pages in search results
- Creating snippets for service pages, forms, departments, topics, events, and news
- Reviewing whether a page title and summary match user intent

## Snippet standards

A useful search snippet should answer:

- What is this page?
- Who is it for?
- What can the user do here?
- How is it different from similar results?

## Title guidance

Page titles should be specific and front-loaded with the user concept or task.

Good patterns:

- Apply for a building permit
- Pay property taxes
- Report illegal dumping
- Request public records
- Register for emergency alerts
- Check eligibility for benefits

Avoid:

- Welcome to...
- Department of...
- Services
- Resources
- Forms
- Information
- Program names without context

## Description guidance

Descriptions should be one or two plain sentences. Include the task, audience, and outcome when relevant.

Pattern:

`Use this page to [task]. Find out [requirements/status/timing/documents] and [next action].`

Alternative pattern:

`Information for [audience] about [topic], including [top user needs].`

## Output format

Provide:

- Recommended page title
- Recommended search result description
- Primary keywords or query terms
- Distinguishing terms for similar pages
- Notes on pages that may compete in search

## Review checklist

- Does the title start with the user task or topic?
- Can users distinguish this page from similar pages?
- Does the snippet avoid vague claims?
- Does it include useful verbs such as apply, pay, report, request, register, appeal, contact, or check?
- Does it avoid internal jargon?
