# Subskill: Form Field Design

## Use this when
Reviewing or creating form fields, labels, sections, required questions, optional questions, validation rules, and input formats.

## Goals

- Reduce errors and incomplete submissions.
- Ask only for information that is necessary.
- Use field labels that match the resident's understanding.
- Group questions in a sequence that follows the resident's task.

## Guidance

### Confirm the need for each field
For each field, ask:

- Why do we need this?
- Who uses it?
- Is it required by law, policy, system processing, or staff review?
- Can it be inferred, prefilled, or collected later?
- What happens if the resident leaves it blank?

Remove or defer fields that are not necessary at this step.

### Use direct labels
Prefer labels that name the information requested.

Better:

- Full name
- Mailing address
- Phone number
- Date of birth
- Property address
- Preferred language

Avoid labels that require agency knowledge:

- Applicant identifier
- Responsible party information
- Service location data
- Supplemental contact details

### Clarify required and optional fields
Do not rely only on visual indicators. Use text when needed:

- Required
- Optional
- Required if you are applying on behalf of someone else
- Required if your mailing address is different from your home address

### Use conditional fields carefully
Only show conditional questions when they apply. If the form is static, make the condition explicit:

- If you answered Yes, provide the date.
- Complete this section only if you are requesting an appeal.
- Skip this section if you already submitted these documents.

### Match input format to the answer
Use the simplest reliable input type:

- Date field for dates
- Radio buttons for one choice
- Checkboxes for multiple choices
- Short text for names and IDs
- Long text only when narrative is needed
- File upload only when attachment is required

### Provide examples for ambiguous fields
Examples should be specific and local where appropriate:

- Example: 3501 Civic Center Drive, San Rafael, CA 94903
- Example: MM/DD/YYYY
- Example: Jane Smith, parent or guardian

## Review checklist

- Each field has a clear purpose.
- Required fields are actually required.
- Labels are plain and specific.
- Conditional fields are clear.
- Input formats match expected answers.
- Error-prone fields include examples or constraints.
- The sequence follows how residents gather and provide information.
