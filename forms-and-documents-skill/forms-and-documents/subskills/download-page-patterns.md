# Subskill: Download Page Patterns

## Use this when
Improving pages that host PDFs, forms, packets, templates, brochures, reports, agendas, or other downloadable documents.

## Goals

- Help residents know which file they need.
- Explain what to do before and after downloading.
- Avoid pages that are only lists of unexplained links.
- Reduce duplicate or outdated downloads.

## Required context for each download
For each file, provide:

- Plain-language title
- One-sentence description
- Who should use it
- What it is used for
- File type and size when useful
- Last updated date when available
- Language version when applicable
- Whether it must be printed, signed, uploaded, emailed, mailed, or brought in person
- Related deadline, fee, or eligibility note if relevant

## Page structure pattern

1. Page heading that describes the resident task
2. Short introduction explaining what the downloads are for
3. Decision help if there are multiple files
4. Download links grouped by task, audience, or document type
5. Instructions for completing and submitting files
6. Contact or help information
7. Related services or pages

## Link label guidance
Use descriptive link text.

Better:

- Download the building permit application
- Download the fee waiver request form
- View the agenda packet for May 19, 2026

Avoid:

- Click here
- Form
- Packet
- PDF
- Download

## Multiple-download guidance
If there are many files, group them by resident need, not internal department structure.

Possible groups:

- Apply for a permit
- Renew a license
- Request an inspection
- Appeal a decision
- Required attachments
- Translated forms
- Reference documents

## Review checklist

- Downloads are not presented as unexplained file lists.
- Each file has a clear purpose and audience.
- Submission instructions are included.
- Outdated, duplicate, and redundant files are flagged.
- HTML alternatives are recommended where appropriate.
- Related pages and next steps are linked.
