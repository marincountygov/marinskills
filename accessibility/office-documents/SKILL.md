# Office Document Accessibility Skill

Use this skill to create, review, or remediate accessible County of Marin Word, Excel, PowerPoint, and similar office documents.

## Purpose

Office documents are often the source files for PDFs, attachments, reports, forms, presentations, and internal communications. Accessibility should be built into the source document before export or publication.

## When to use this skill

Use this skill for:

- Word documents.
- Excel spreadsheets.
- PowerPoint presentations.
- Google Docs, Sheets, and Slides.
- Templates.
- Internal documents that may later become public.
- Source files used to generate PDFs.

## General accessibility requirements

Office documents should:

- Use built-in headings and styles.
- Use logical reading order.
- Include descriptive document titles.
- Use meaningful link text.
- Provide alt text for informative images.
- Mark decorative images appropriately where supported.
- Use sufficient color contrast.
- Avoid using color alone to communicate meaning.
- Use accessible tables with clear headers.
- Avoid merged cells where they interfere with comprehension or assistive technology.
- Avoid images of text unless essential.
- Use plain language and clear structure.
- Include contact information for help when appropriate.

## Word document requirements

Word documents should:

- Use heading styles in order.
- Use real lists, not manually typed bullets or numbers.
- Use simple tables with header rows.
- Avoid text boxes when they disrupt reading order.
- Use descriptive file names.
- Include language metadata.

## Spreadsheet requirements

Spreadsheets should:

- Have clear sheet names.
- Avoid blank rows and columns inside data tables.
- Identify table headers.
- Avoid merged cells in data regions.
- Include notes explaining abbreviations, formulas, and data sources.
- Use charts with clear labels and text alternatives.
- Avoid using color as the only indicator of status or meaning.

## Presentation requirements

Presentations should:

- Use accessible slide layouts.
- Give every slide a unique title.
- Confirm reading order on each slide.
- Use large, legible text.
- Avoid dense slides.
- Provide alt text for meaningful visuals.
- Ensure charts and diagrams have text equivalents.
- Provide accessible handouts or notes when needed.

## Review workflow

1. Check document structure and styles.
2. Review headings, lists, tables, links, images, and reading order.
3. Run the built-in accessibility checker.
4. Manually verify issues the checker cannot detect.
5. Confirm plain-language and brand alignment.
6. If exporting to PDF, verify the exported PDF separately.

## Output format

When reviewing an office document, provide:

- Accessibility status.
- Blocking issues.
- Recommended fixes.
- Source-document improvements before export.
- Plain-language and brand notes.
